# 写作对象研究协议

> 用于学术作者 skill 在写新文章前补足对象事实。核心原则：事实来自检索和核验，论证方式来自作者蒸馏。

## 一、何时必须做对象研究

以下对象默认需要先检索或本地核验，再进入作者模型写作：

- 技术、项目、软件、平台、模型、协议、开源仓库。
- 法律、政策、监管规则、司法案例、行政执法事件。
- 公司、机构、组织、产品线、商业模式。
- 新近事件、争议、事故、舆情、行业变化。
- 法律风险主题，例如数据安全、隐私、版权、权限、身份认证、专业责任。

只有用户明确要求“只用本地材料”或“不要联网”时，才跳过网络检索；此时必须写明资料边界和可能过时之处。

## 二、来源优先级

### 技术/项目对象

1. 官方文档、官方 CLI reference、官方配置说明。
2. GitHub README、release、issue、security advisory。
3. 本机真实状态，例如 `--help`、`status`、配置、日志、版本、运行端口。
4. 涉及法律评价时，进入专业知识库/私有知识库检索相关论文和学术观点。
5. 权威安全报告、学术论文、可信媒体。
6. 社交媒体、论坛、博客只能作为线索，不作为核心事实。

### 制度/政策对象

1. 法律法规、监管机构文件、司法解释、政策原文。
2. 官方问答、实施指南、监管通报、裁判文书。
3. 专业知识库/私有知识库中的学术论文、综述、评论和争点材料。
4. 权威评论、行业报告。
5. 新闻报道只作背景，不能替代规范文本。

### 人物/组织对象

1. 官网、机构介绍、公告、年报、监管披露。
2. 公开访谈、正式演讲、署名文章。
3. 可信新闻与同行评价。
4. 百科和不可核验转述只作弱线索。

### 案例/事件对象

1. 官方通报、裁判文书、监管处罚、当事方公告。
2. 多来源时间线交叉核验。
3. 区分已确认事实、争议事实和推断。

### 专业知识库/私有知识库层

法学、社科、政策和法律风险写作默认需要检索用户可访问的专业知识库。若使用腾讯 ima，先搜索知识库名称，再进入目标知识库检索论文；数字法学主题默认搜索「数字法学」并优先选择「数字法学研究」，但不要把某次发现的知识库 ID 写成唯一入口。

知识库检索策略：

1. 先用对象名称检索，再用“对象名称 + 法律问题”检索。
2. 技术对象加入风险词：数据、个人信息、算法责任、著作权、证据、网络安全、权限、专业责任。
3. 制度对象加入规范词：治理、监管、责任、权利、程序、合规、法治。
4. 案例事件加入争点词：裁判理由、责任承担、证明、损害、平台、监管。
5. 至少记录论文题名、作者、可见摘要/高亮/片段、可用观点和无法读取全文的边界。

知识库结果的定位：

- 知识库论文提供文献观点、理论争点、概念资源和研究缺口。
- 知识库论文不能替代法律法规、监管文件、司法案例、官方文档或本地状态。
- 如果知识库不可访问，必须列出拟检索关键词和资料缺口；不能假装完成了文献阅读。

## 三、研究档案结构

对象研究完成后，写入或临时形成 `object-research-dossier.md`：

```markdown

- 对象类型：
- 写作问题：
- 检索日期：

## 来源清单
| 来源 | 类型 | 用途 | 可信度 | 时间 |
| --- | --- | --- | --- | --- |

## 事实层
- 技术/制度结构：
- 关键行为链：
- 版本、时间和场景：
- 利益相关方：

## 学术文献层
- 知识库名称：
- 检索关键词：
- 已读论文条目：
- 摘要/高亮/片段：
- 可用学术观点：
- 文献边界：

## 法律风险层
- 数据与个人信息：
```

## 四、进入作者模型前的判断

对象研究结束后，再决定如何调用作者模型：

- 事实层不能由作者风格补齐。
- 文献层不能由网络新闻或作者风格补齐。
- 作者模型只负责问题意识、概念重构、论证框架和表达节奏。
- 引文、法条、案例、版本号和技术细节必须来自研究档案或用户材料。
- 如果对象事实或文献观点不足，应先列出补证清单，而不是直接生成完整论文。

## 五、质量门

合格的对象研究至少满足：

- 有三个以上来源类型，或说明为什么无法取得更多来源。
- 对技术对象有结构和行为链描述。
- 对法律对象有规范来源和适用边界。
- 对法学/社科写作有专业知识库或学术文献检索记录，或明确说明知识库不可用。
- 对风险主题有不少于五类风险维度。
- 明确区分事实、推断、文献观点、作者式论证。
