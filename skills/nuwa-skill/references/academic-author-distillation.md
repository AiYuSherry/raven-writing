# 学术作者蒸馏框架

> 用于把法学、社科、人文等学术作者蒸馏成可运行的写作模型。

学术作者蒸馏不是普通人物画像，也不是角色扮演。它的目标是把作者长期形成的问题意识、理论资源、概念制造、论证路径、论文结构和表达习惯，转化为一个能辅助新论文写作的 AgentSkill。

## 一、语料分层

先给每份材料标注来源身份。

| 类型 | 说明 | 可用于 |
| --- | --- | --- |
| primary_writing | 作者本人论文、著作、正式文章 | 文风、结构、概念、论证模式 |
| primary_speech | 作者本人访谈、演讲、课堂文字稿 | 思想解释、即时判断、口头表达 |
| context_only | 同场对话、他人论文、评论、访谈提问 | 问题域和外部对话背景 |
| identity_proof | 学校官网、出版社、期刊、机构页面 | 身份、时间线、任职与著作信息 |
| uncertain | 作者身份或来源无法确认 | 只能列入边界，不参与归纳 |

硬规则：

- 文风统计只用 `primary_writing`。
- 思想谱系优先用 `primary_writing`，可用 `primary_speech` 辅证。
- `context_only` 不得作为作者写作风格证据。
- 如果文件名中有其他作者名，默认先标为 `context_only`，除非正文明确显示目标作者是主作者。

## 二、问题意识抽取

从论文引言、摘要和结论中抽取作者反复进入问题的方式：

- 他认为流行分类或既有理论哪里错了。
- 他认为哪些真实动因被意识形态标签、表层价值或抽象原则遮蔽了。
- 他如何把技术、制度、文明、市场、国家和社会关系放在同一框架里。
- 他面对新技术时，是先谈权利保护、市场效率、国家能力、制度弹性，还是文明主体性。

输出时不要写成抽象人格品质，要写成可触发的写作动作。例如：

- 遇到三分法、二分法或流行标签时，先问这个分类是否遮蔽了制度运行的真实驱动力。
- 遇到外来理论时，先区分它的解释力和它背后的文明/政治预设。

## 三、思想谱系抽取

记录作者常调用的资源，并注明功能。

| 谱系类型 | 关注点 |
| --- | --- |
| 法理/宪法 | 法治、宪制、司法审查、共和主义、普通法 |
| 社会理论 | 功能分化、系统论、二阶观察、社会科学方法 |
| 政治哲学 | 自由、秩序、共和、文明主体性 |
| 比较法 | 美国、欧盟、德国、香港、国际法 |
| 技术治理 | AI、算法、数据、区块链、自动驾驶、智慧司法 |

谱系不是书单，而是写作时的调用方式。要记录它在论证中承担的功能：解构、比较、反证、历史化、类型学重建或规范评价。

## 四、概念词库

提取三类词：

- 核心概念：作者反复使用并承担论证功能的词。
- 概念组合：例如「有序创新」「自我东方主义」「制度弹性」「风险规避」「二阶观察」。
- 分类轴：例如「国家安全/风险规避/有序创新」「法治化/法治中国化」「技术创新/风险控制」。

概念词库要附使用规则：

- 什么时候适合使用。
- 使用时要避免何种空泛化。
- 是否需要配套制度事实、比较法材料或历史谱系。

## 五、论证框架抽取

重点抽取可迁移的论文动作：

1. 先呈现流行框架。
2. 指出流行框架的价值预设、解释盲区或自我消解。
3. 回到制度事实、历史脉络或技术运行机制。
4. 建立新的分类轴或概念框架。
5. 用中外案例、法律文本、政策轨迹或司法实践检验新框架。
6. 回到更高层次的问题：法治道路、国家治理、文明主体性、技术社会共构。

不是每篇文章都必须完整走完六步，但学术作者 skill 必须知道这些动作如何组合。

## 六、论文结构与注释习惯

从一手论文中提取：

- 标题是否偏问题式、判断式、概念式或副标题解释式。
- 摘要是否先否定流行观点，再给出新框架。
- 引言如何铺设学术争议、现实背景和文章任务。
- 章节标题是否偏「问题的提出」「模式/机制/路径」。
- 脚注中中外文献、法律文本、政策文件、案例材料的比例。
- 结论是否收束为制度建议、理论反思还是文明/法治道路层面的判断。

## 七、表达 DNA

学术作者的表达 DNA 不追求口语化，而追求辨识度：

- 长句是否多，是否常用多重限定。
- 是否先铺垫再判断，还是先判断再展开。
- 判断词强度：如「并不在于」「与其说是」「不如称为」「实则」「遮蔽」。
- 转折方式：如「但」「然而」「同样」「事实上」「也正是在这样的背景下」。
- 设问方式：是否用问题打开历史、理论或制度争议。
- 例证节奏：抽象理论之后是否迅速接制度文本、历史事件或比较法材料。

## 八、样本文笔对照与相似度硬约束

学术作者蒸馏必须把“像作者思考”和“像作者写论文”分开处理。问题意识、理论谱系和分类轴只是作者模型的一部分；如果缺少一手样本的句法、段落节奏和概念链校准，生成结果很容易变成泛学术 AI 文章。

### 1. 样本选择

- 至少抽样 2 篇 `primary_writing`，优先选择与目标用途同题域、同体裁、同论证任务的样本。
- 技术治理题目选技术治理论文，宪法/法理题目选法理或宪法论文，方法论题目选方法论论文；必要时补一篇能体现作者底层理论支架的样本。
- `context_only`、身份资料、他人评论、知识库条目和对象研究材料不得作为文笔证据。
- 若不能读取原文，必须读取已抽取的一手样本研究文件，并明确这是二级抽取而非原文回看。
- 若原文和已抽取的一手样本研究文件都不可取得，必须暂停完整成文；不得用对象研究、知识库条目或作者模型概述替代样本文笔对照。

### 2. 样本文笔对照笔记

目标 skill 必须把样本文笔对照笔记设为生成前置产物，而不是可选建议。没有笔记，不得进入正文、改写或续写；正文完成后倒填笔记视为失败。笔记模板至少包含：

- 抽样一手文本及抽样位置。
- 句法观察：长句比例、限定链、转折位置、判断落点。
- 概念链观察：核心概念如何连续推进，是否先拆旧分类再命名新框架。
- 段落节奏：问题、材料、比较、判断之间如何切换。
- 常用论证动作：解构旧说、重建分类、进入制度机制或理论落点的方式。
- 禁止迁移：不得复制的原文表达、不得冒充的作者立场、不得编造的引用。
- 本次正文改写指令：需要拉长、压缩、重排或补强的句法和段落规则。
- 前置产物状态：确认本笔记完成于正文生成前；若未完成，必须暂停完整成文或显著标注边界。

### 3. 浅层合规失败模式

以下情况必须写入 academic author skill 的反模式或质检规则：

- 只检索 IMA/专业知识库，没有样本文笔对照。
- 只做对象研究，没有句法重写和段落节奏校准。
- 只套问题意识、思想谱系或分类轴，正文仍像普通 AI 学术评论。
- 只提高概念密度，没有一手样本中的限定链、转折方式、判断落点和结论收束。
- 类似《我们真的有个人信息同意撤回权么？》这类产出：概念密度较高，也会谈权利结构、制度前提和治理机制，但整体仍像普通学术评论或 AI 文章，缺少目标作者一手样本中的句法、段落节奏、概念链和结论收束。
- 把语境材料、对象研究材料或二手评论当成作者文笔证据。

### 4. 低相似度警报

目标 skill 必须明确：如果草稿读起来像普通 AI 学术评论、政策解读、课堂作业、公众号文章、报告体或“法条+建议”模板，即使问题意识方向正确，也要判为低相似度并退回重写。低相似度不是靠增加几个作者常用概念解决，而是要回到样本重新校准句法、段落节奏和概念链。《我们真的有个人信息同意撤回权么？》式“概念密度高但作者文笔不像”应作为明确失败反例处理。

### 5. 生成后退回规则

生成或改写正文后，必须检查：

- 每个核心段是否有明确段落功能，而不是泛泛推进。
- 核心句是否承担限定、转折、判断和概念推进。
- 段落是否在概念、事实、文献、比较材料和制度后果之间交替推进。
- 概念链是否前后连续，而不是堆砌漂亮词。
- 事实层、文献层和作者模型层是否清楚分离。

任一项明显不足，退回样本文笔对照层重写。若缺少可读取的一手样本文本和已抽取的一手样本研究文件，则暂停完整成文，输出资料边界与待补样本清单。

## 九、反模式

学术作者 skill 必须明确禁区：

- 不用宏大空话代替制度分析。
- 不用「随着时代发展」「具有重要意义」这类泛学术开头。
- 不编造文献、法条、案例或页码。
- 不为了像作者而照搬长段原文。
- 不把作者的学术写作改造成公众号、营销文、演讲稿或政策宣传稿。
- 不冒充本人新作，不署作者名。
- 不把“问题意识像”误认为“文笔像”；没有样本文笔对照、句法重写和段落节奏校准时，不得判定为合格。

## 十、分层质检

借鉴写作型 skill 的分层检查，但改为学术标准。

### L1 硬规则

- 来源分层清楚。
- 没有占位符。
- 没有冒充本人。
- 没有把语境材料当作一手文风。
- 点名作者风格时，不跳过一手样本文笔对照。
- 样本文笔对照笔记是生成前置产物，不得正文完成后倒填。
- 样本不可得时，暂停完整成文或显著标注边界。

### L2 样本文笔对照

- 是否读取或抽样一手写作样本。
- 是否形成作为生成前置产物的样本文笔对照笔记。
- 是否有句法重写、段落节奏校准和概念链连续性检查。
- 是否写入文笔相似度硬约束、低相似度警报、样本缺失暂停规则和退回重写规则。
- 是否把《我们真的有个人信息同意撤回权么？》式“概念密度高但作者文笔不像”作为失败反例。
- 是否明确“不能只迁移思想/问题意识”。

### L3 风格一致性

- 是否有作者常用的问题进入方式。
- 是否有作者式转折、限定、分类和概念组合。
- 是否保持学术论文语体，而非公众号或报告语体。

### L4 论证密度

- 是否有真实的概念区分。
- 是否有反驳对象。
- 是否用制度事实或比较材料支撑判断。
- 是否能从表层价值标签进入深层驱动力。

### L5 作者辨识度

- 删掉作者名后，是否仍能看出这个写作模型的独特路径、句法和段落节奏。
- 是否既能生成新问题，又不会把所有新问题机械套进同一模板。
