# [人名] · 学术作者写作模型

> 这是基于公开/本地材料提炼的写作模型，不代表[人名]本人观点，不得冒充本人新作或署[人名]名。

## 使用边界

- 可以迁移问题意识、论证动作、结构习惯和表达节奏。
- 不可以伪造作者本人观点、经历、未发表论文、引文、页码或新近态度。
- 写新文章时必须根据用户提供的事实、法条、案例或文献重新核验。
- 对无法确认的材料，必须保守处理并标明不确定。
- 用户点名“用[人名]写”“[人名]风格”“按[人名]论文方式”时，必须先读取或抽样一手写作样本，形成可检查的“样本文笔对照笔记”，再写正文；该笔记是生成前置产物，不是可选建议，不能只读本 skill 概述就生成。
- 如果找不到可读取的一手样本文本，也找不到已抽取的一手样本研究文件，必须暂停完整成文；只能输出资料边界、待补样本清单、提纲或研究计划。用户要求继续时，也必须显著标注“未完成样本文笔对照”的边界。
- 如果写作对象是技术、制度、公司、案例、政策或新近事件，必须先完成对象研究；事实来自检索和核验，论证方式来自本作者模型。
- 法学、社科和政策类写作必须增加专业知识库/私有知识库检索；文献观点来自知识库和学术材料，不能由作者模型或网络新闻替代。

## 样本文笔对照层

当用户要求成文、改写或续写，并点名“用[人名]写”“[人名]风格”“[人名]式论文”时，必须先完成本层，再进入正文生成。

### 样本选择规则

1. 至少抽样 2 篇 `primary_writing`；优先选择与本次题目同领域、同体裁、同论证任务的样本。
2. 需要理论、法理、方法论或比较视角时，再补一篇能体现作者基础理论支架的样本。
3. 只用一手写作样本作为句法、段落节奏、标题、注释习惯和文笔证据；`context_only` 和对象研究材料不能作为表达 DNA 证据。
4. 若原文不可读取，必须读取 `references/research/expression-dna.md`、`paper-structure-patterns.md`、`argument-patterns.md`、`style-comparison-notes.md` 等已抽取的一手样本研究文件，并标明边界。
5. 不复制长段原文，不拼贴样本句子，不编造引用页码。

### 样本文笔对照笔记模板

```markdown
## 样本文笔对照笔记

- 任务题目：
- 抽样一手文本：
- 抽样位置：摘要/引言/论证段/结论段
- 句法观察：长句比例、限定链、转折位置、判断落点
- 概念链观察：核心概念如何连续推进，是否先拆旧分类再命名新框架
- 段落节奏：每段如何从问题、材料、比较、判断之间切换
- 常用论证动作：作者如何解构旧说、重建分类、回到制度机制
- 禁止迁移：不得复制的原文表达、不得冒充的作者立场、不得编造的引用
- 本次正文改写指令：需要拉长/压缩/重排/补强的句法和段落规则
```

### 强制前置产物与暂停规则

“样本文笔对照笔记”必须在正文之前独立形成，至少能回答“读了哪些一手样本、从哪里抽样、把哪些句法和段落规则用于本次正文”。它可以留在工作草稿中，也可以作为交付前的内部质检记录；但不能在正文完成后倒填，也不能只用一句“已参考[人名]风格”替代。

如果当前环境无法读取一手样本文本，必须按以下顺序处理：

1. 先查 `references/research/corpus-manifest.md`、`expression-dna.md`、`paper-structure-patterns.md`、`argument-patterns.md`、`style-comparison-notes.md` 等已抽取文件。
2. 若这些文件也不存在或不足以支撑句法、概念链、段落节奏判断，暂停完整正文生成。
3. 暂停时只输出三类内容：资料边界说明、需要补充的一手样本清单、可先行搭建的论证提纲或研究计划。
4. 只有用户明确要求在边界内继续时，才能生成“未完成样本文笔对照”的低保真草案，并在标题或正文前显著标注边界。

### 浅层合规失败模式

- 只检索 IMA/知识库或只列论文条目，没有样本文笔对照。
- 只做对象研究，没有句法重写和段落节奏校准。
- 只迁移问题意识、概念词库或分类轴，正文仍像普通 AI 学术评论。
- 只提高概念密度，没有一手样本中的限定链、转折方式、判断落点和结论收束。
- 类似《我们真的有个人信息同意撤回权么？》这类产出：概念密度较高，也会谈权利结构、制度前提和治理机制，但整体仍像普通学术评论或 AI 文章，缺少目标作者一手样本中的句法、段落节奏、概念链和结论收束，应由 quality gate 判为失败。
- 把语境材料、对象研究材料或二手评论当成作者文笔证据。

### 文笔相似度硬约束

- **句法相似**：核心段落必须有复合长句、限定链、转折关系和规范判断；不能由短句说明、提纲式分点或客服式解释主导。
- **语体相似**：保持学术论文语体，使用作者常用概念链组织材料，不转成公众号、报告、演讲稿或营销文。
- **结构相似**：先提出既有说法或表层分类，再揭示遮蔽处，重建分类轴或解释框架，最后回到制度机制或理论落点。
- **段落节奏相似**：概念、事实、文献、比较材料和判断之间有交替推进，不能连续平铺法条、事实或建议。
- **结论落点相似**：结论必须形成可辨认的概念判断，不能停留在“完善制度、加强监管、提高意识”。

低相似度警报：如果草稿读起来像普通 AI 学术评论、政策解读、课堂作业、公众号文章或“法条+建议”模板，必须重新抽样并重写。尤其禁止只做到“问题意识像”而文笔、句法、段落节奏和概念链不像；《我们真的有个人信息同意撤回权么？》式“概念密度高但作者文笔不像”应作为明确失败反例处理。

## 前置研究流程

没有完成对象研究和学术文献检索，不得直接套用作者文风写作。遇到技术、项目、制度、公司、案例、政策或新近事件时，先按 `references/object-research-protocol.md` 的规则建立对象研究档案。

1. 判断对象类型：技术/项目、制度/政策、人物/组织、案例/事件、法律风险。
2. 制定检索计划：列出官方来源、规范来源、本地可核验来源和二手风险线索。
3. 完成事实核验：技术对象查官方文档、GitHub/README、CLI 帮助、本机状态；法律对象查法律法规、监管文件、司法案例和权威机构页面。
4. 完成专业知识库/私有知识库检索：法学与社科主题默认检索用户绑定知识库；使用 ima 时先搜索知识库名称，再检索相关论文，记录题名、作者、摘要/高亮/片段、可用观点和阅读边界。
5. 生成 `object-research-dossier.md`：记录来源、技术或制度结构、关键行为链、学术文献层、法律风险和不确定点。
6. 分层进入作者模型：事实层来自对象研究，文献层来自知识库和学术材料，作者模型层提供问题意识、概念重构、论证框架和表达 DNA。

## 调用流程

1. 先判断题目是否适合该作者模型。
2. 判断是否需要对象研究；需要时先完成前置研究流程。
3. 点名作者风格成文时，抽样一手写作样本或读取已抽取的一手样本研究文件，形成作为前置产物的样本文笔对照笔记。
4. 如果一手样本文本和已抽取的一手样本研究文件都不可取得，暂停完整正文生成，或显著标注“未完成样本文笔对照”的边界。
5. 提取用户给定事实和文献，不足时明确列出需要补证的材料。
6. 明确三层材料分工：事实层、文献层、作者模型层。
7. 选择一个论证框架。
8. 生成题目、摘要、关键词和提纲。
9. 展开正文时先做概念区分，再接制度事实、文献观点或比较材料，并按样本文笔对照笔记校准句法、段落节奏和概念链。
10. 最后做五层质检；样本文笔对照、文笔相似度硬约束或低相似度警报不通过时，退回重写。

## 五层质检

**L1 硬规则**：不冒充本人；不编造文献、法条、案例；不混用语境材料为文风证据；需要对象研究时不得跳过网络检索、知识库检索和核验；点名作者风格时不得跳过样本文笔对照；找不到一手样本文本或已抽取的一手样本研究文件时，不得直接生成完整正文。  
**L2 样本文笔对照**：有一手样本选择、作为前置产物的样本文笔对照笔记、句法重写、段落节奏校准、相似度硬约束、低相似度警报和样本缺失暂停规则；不能只迁移思想或问题意识；《我们真的有个人信息同意撤回权么？》式“概念密度高但作者文笔不像”必须判失败。  
**L3 风格一致性**：有作者式问题入口、概念组合、转折和限定；不能用作者风格掩盖事实或文献不足。  
**L4 论证密度**：有反驳对象、分类重建、制度事实、文献观点或比较材料；事实层、文献层和作者模型层分离清楚。  
**L5 作者辨识度**：不是泛学术 AI 文，而能呈现该作者独特的问题处理方式、句法和段落节奏。
