# 写作对象研究档案规则

本文件不是某一次写作的事实档案，而是郑戈学术作者模型每次处理新对象前必须执行的研究结构。具体写作时，应根据对象另建临时档案或在回答开头列出研究计划与来源类型。

## 触发对象

- 技术系统：OpenClaw、Codex、浏览器代理、AI agent、开源项目、自动化平台。
- 法律制度：法律法规、监管政策、司法案例、行政执法、合规规则。
- 组织与产品：公司、机构、产品线、平台治理、商业模式。
- 新近事件：事故、争议、舆情、行业变化、版本更新。
 
OpenClaw 只是技术系统的一种验收样例。本规则面向所有写作对象，不能把对象研究能力理解为 OpenClaw 专用。

## 事实来源优先级

1. 官方文档、官方 CLI reference、官方配置说明。
2. GitHub/README、release、issue、security advisory。
3. 本机真实状态，例如 `--help`、`status`、配置、日志、版本、运行端口。
4. 法律法规、监管文件、司法案例、权威机构页面。
5. 专业知识库/私有知识库论文，例如用户 ima 中的「数字法学研究」。
6. 权威媒体、安全报告、行业报告。
7. 论坛、社交媒体、博客仅作风险线索。

## 学术文献层

凡是法学、社科、政策或法律风险写作，事实核验之后必须进入学术文献层。默认流程：

1. 使用 ima 搜索知识库名称，数字法学主题优先检索「数字法学」并选择「数字法学研究」。
2. 在知识库内用“对象名称 + 法律问题”检索相关论文。
3. 记录论文题名、作者、可见摘要/高亮/片段、可用观点、理论争点和无法读取全文的边界。
4. 将文献观点转化为可供郑戈模型处理的争点，而不是把论文条目堆成文献综述。

知识库论文只提供文献观点和学术谱系，不能替代规范原文、官方事实、本地核验或对象技术结构。

## 法律风险谱系

- 数据与个人信息。
- 权限、授权与越权行为。
- 渠道、身份识别与外部输入。
- 持久记忆、档案、记录和数据保留。
- 插件、依赖、供应链与第三方服务。
- 日志、证据与可追责性。
- 著作权、内容生成与引用边界。
- 网络安全、接口开放与远程访问。
- 专业责任、组织授权与内部治理。

## 对象类型矩阵

- 技术系统：架构、模块、权限、数据流、接口、版本、依赖、插件、日志、安全说明。
- 法律制度：规范文本、制定机关、适用范围、义务主体、权利结构、责任后果、实施时间。
- 公司组织：官网公告、商业模式、治理结构、用户协议、隐私政策、监管披露、争议事件。
- 司法案例：裁判文书或处罚决定、事实认定、争点、裁判理由、法律依据、程序节点。
- 新近事件：时间线、已确认事实、争议事实、各方说法、权威通报、证据缺口。

## 进入郑戈模型的转换

先完成对象研究和学术文献检索，再进行五步转换：

1. 明确三层材料分工：事实层、文献层、作者模型层。
2. 拆解流行说法。
3. 重建分类轴。
4. 把技术事实和文献争点转化为法律问题。
5. 回到法治秩序、制度弹性和责任结构。
